Why malloc doesn't malloc

发表于 2023-10-11

从一个简单的C程序开始

这里有一个简单的程序将会调用malloc请求指定数目的Megabytes.

但当你运行程序 (例如，./a.out 1000) 后有意思的事情就发生了

我们可以看到，内存仅使用了649MB?! 仅这个程序而言，我们就申请了1GB的内存，但整个系统仅使用了近600MB?

Here's the man page of free for ref.

更有力的工具: pmap

free 提供的信息少的可怜，我们来看看pmap能揭示的信息有哪些

可以看到 0000564218248000H 处有近1GB的虚拟地址空间。但往右处一瞥，我们发现实际的内存只占了12KB!

一行代码让OS给我开辟1G空间

在我们尝试理解这其中的奥秘前，先来看看一个稍微改动的之前的小C程序

与之前不同的是，我们将for循环里的语句换成了赋值语句。再来看看free提供的信息。 以及pmap提供的更全面的信息

我们可以看到，两者都输出了近1GB的内存使用空间。

匿名文件

一个值得注意的点是，pmap标注了我们申请的区域为[anon]，这意味着我们申请的内存对应着的是匿名文件。一个相似的比喻是ELF格式中的.bss区域。这意味着该文件为假想的页，实际上并不占用存储空间，当我们访问到其中的区域时，我们所得到的值必然为0。当我们写时，导致匿名文件的值不再为0，因此需要占用内存资源存放信息。这也就是为什么仅仅读写之分便可消耗大量的内存空间。

  2. 匿名文件的管理应由操作系统完成，还是由malloc完成？

Independence and Suggestion

发表于 2023-10-10 更新于 2023-10-11

Independence

Page 73, Problem 1.43.

The events \(A\) and \(B^c\) are independent if \(A\) and \(B\) are independent. \[\begin{aligned} P(B)P(A) + P(A\cap B^c) = P(A) = P(B)P(A) + P(B^c)P(A) \end{aligned} \] The first equality holds because \[\begin{aligned} P(A\cap B) + P(A\cap B^c) = P(A) = P(B)P(A) + P(A\cap B^c) \end{aligned}\]

Transibility of Independence on set's complement

The events \(A^c\) and \(B^c\) are independent if \(A\) and \(B\) are independent. Apply the previous result on \(A\) and \(B\), then on \(B^c\) and \(A\).

Suggestion

Page 69, Problem 1.29.

Let \(A\) and \(B\) be events with \(P(A) > 0\) and \(P(B) > 0\). We say that an event \(B\) suggests an event \(A\) if \(P(A|B) > P(A)\), and does not suggest event \(A\) if \(P(A|B) < P(B)\).

Suggestion is mutual

\[\begin{aligned} P(A|B) > P(A) \iff P(B|A)> P(B) \end{aligned} \]

\[\begin{aligned} P(A|B) &> P(A)\\ P(A\cap B)&>P(A)P(B) \end{aligned}\] Notice the symmetry in the above equivalent formula.

Suggestion is complementary

\[\begin{aligned} P(A|B) > P(A) \iff P(A|B^c) < P(A) \label{eq:complement} \end{aligned}\]

\[\begin{aligned} P(B)P(A) + P(B^c)P(A) = P(A)= P(B)P(A | B) + P(B^c)P(A | B^c) \end{aligned}\] If \(P(A|B) > P(A)\), to make the above equality holds, \(P(A)\) must be less than \(P(A|B^c)\).

Notice [eq:complement] could be rewritten as \[\begin{aligned} P(A|B) > P(A) > P(A|B^c), \end{aligned}\] if \(A,B\) suggest each other. Thus, the following argument is always true \[\begin{aligned} P(A|B) > P(A|B^c) \iff P(A|B) > P(A) \end{aligned}\]

Suggestion is not only complement on the condition, but also on the conditional event, i.e., \[\begin{aligned} P(A|B) > P(A) \iff P(A^c|B) < P(A^c) \end{aligned}\]

\[\begin{aligned} P(A|B) + P(A^c|B) = 1= P(A) + P(A^c) \end{aligned}\]

Double complement suggestion have the same result as the original. In other words, \[\begin{aligned} P(A|B) > P(A) \iff P(A^c|B^c) > P(A^c) \end{aligned}\] This equivalence is trivial by the above arguments.
